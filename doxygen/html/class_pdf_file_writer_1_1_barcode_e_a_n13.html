Detailed Description

Barcode EAN-13 or UPC-A class

Barcode EAN-13 or UPC-A Note UPC-A is a subset of EAN-13 UPC-A is made of 12 digits EAN-13 is made of 13 digits If the first digit of EAN-13 is zero it is considered to be UPC-A. The zero will be eliminated. The barcode in both cases is made out of 12 symbols.

PdfFileWriter.BarcodeEAN13.BarcodeEAN13 ( String  Text)

Barcode EAN13 Constructor

Parameters
TextInput text

Convert text to code EAN-13 or UPC-A.

All characters must be digits.

The code is EAN-13 if string length is 13 characters and first digit is not zero.

The code is UPC-A if string length is 12 characters or string length is 13 and first character is zero.

The last character is a checksum. The checksum must be given, however the constructor calculates the checksum and override the one given. In other words, if you do not know the checksum just set the last digit to 0.

PdfFileWriter.BarcodeEAN13.BarcodeEAN13 ( Int32[]  _CodeArray)

Barcode EAN13 constructor.

Parameters
_CodeArrayCode array input.

The constructor sets CodeArray and converts it to text.

CodeArray must be 12 elements long for both EAN-13 or UPC-A.

In the case of UPC-A the 12 elements of code array correspond one to one with the 12 digits of the encoded value.

In the case of EAN-13 the 12 code elements corresponds to element 2 to 13 of the text characters. The first text character controls how elements 2 to 5 of the code array are encoded. Please read the following article for full description. http://www.barcodeisland.com/ean13.phtml.

In this class, odd parity encoding is one code element equals one digit.

Even parity is code element equals digit plus 10.

The last code element is a checksum. The checksum must be given however the constructor calculates the checksum and override the one given. In other words, if you do not know the checksum just set the last element to 0.
